## Internet Network
### OSI 7계층
OSI 참조 모델은 컴퓨터 네트워크 프로토콜 디자인과 통신을 계층으로 나누어 설명한 것.
물리적 데이터를 송수신하는 물리계층부터 사용자 애플리케이션 서비스 제공하는 응용계층까지 네트워킹 시스템 구조를 위한 통합 표준을 위해 만들어짐.

1. 물리 계층
    - 데이터 전기적인 신호로 변환해서 주고받는 기능을 진행하는 공간. 
    - 역할 : 하드웨어 명세 정의, 인코딩
    - 데이터 전송 단위 : 비트
    - 리피터, 케이블, 허브

2. 데이터 링크 계층
    - 역할 : 물리 계층으로 송수신되는 정보를 관리하여 안전하게 전달되도록 도와줌.
    - CRC 기반의 오류 제어와 흐름 제어가 필요한 계층. 
        - 포인트 투 포인트 간 신뢰성 있는 전송을 보장해야하기 때문.
    - 논리적 연결 제어(LLC, Logical Link Control), 매체 접근 제어(MAC, Media Access Control)
    - Mac 주소를 통해 통신. 
    - 데이터 전송 단위 : 프레임
    - 대표 프로토콜 : 이더넷, HDLC
    - 브릿지, 스위치

3. 네트워크 계층
    - 역할 : 데이터를 목적지까지 가장 안전하고 빠르게 전달하는 기능을 담당. 다양한 길이의 데이터를 네트워크들을 통해 전달하고, 그 과정에서 전송 계층이 요구하는 서비스 품질(QoS)을 제공하기 위한 수단을 제공.
    - 라우팅, 흐름 제어, 세그멘테이션(segmentation/desegmentation), 오류 제어, 인터네트워킹(Internetworking) 등을 수행
        - 흐름제어? 수신측과 송신측의 데이터 처리 속도 차이 를 해결하기 위한 기법. 송신측의 전송량이 수신측의 처리량보다 많은 경우, 전송된 패킷은 수신측의 큐를 넘어서 손실될 수 있기 때문에 송신측의 패킷 전송량을 제어함.
            - 예시? 정지-대기 (Stop and Wait), 슬라이딩 윈도우 (Sliding Window), Piggyback 방식
                - 정지-대기? 매번 전송한 패킷에 대해 응답을 받아야만 그 다음 패킷을 전송
                    - 장단점? 구조가 간단한 대신, 하나씩 주고받기 때문에 비효율적
                - 슬라이딩 윈도우? 수신측에서 설정한 윈도우 크기만큼 송신측에서 확인 응답없이 세그먼트를 전송할 수 있게 하여 데이터를 동적으로 조절하는 기법. Stop and Wait 기법의 비효율성을 개선한 기법.
                    - 장점? ACK 프레임을 수신하지 않더라도 여러 개의 프레임을 연속적으로 전송가능
                - Piggyback? 수신측에서 수신된 데이터에 대한 확인 응답을 즉시 보내지 않고, 별도의 제어 프레임도 사용하지 않은 채, 전송할 데이터가 있는 경우에만, 그 데이터 프레임 안에 확인 필드를 덧붙여 전송하는 기법
        - 오류제어? 송신측에서 수신측에 전달된 프레임에서 발생한 오류를 감지하고 수정하기 위한 매커니즘
            - 방법? 패리티 검사, 순환 중복 코드(CRC), 체크섬
                - 패리티 검사? 프레임에 포함된 1 비트 수가 짝수인지 홀수인지를 나타내는 단일비트를 프레임에 추가하여 오류 감지. 짝수 개의 오류가 발생한다면 감지 불가
            - 예시? Stop and Wait ARQ, Go-Back-N ARQ, Selective-Reject ARQ
                - Stop and Wait ARQ?
                - Go-Back-N ARQ?
                - Selective-Reject ARQ?
        - 라우팅? 라우팅은 네트워크에서 경로를 선택하는 프로세스. 네트워크가 정체 없이 최대한 많은 용량을 사용할 수 있도록 데이터 트래픽을 관리함으로써 네트워크 통신의 효율성을 높임.
        - 세그멘테이션? 네트워크를 여러 개의 세그먼트나 서브넷으로 나누는 아키텍처 방식으로 각각이 소규모 네트워크 역할을 함. 
            - 왜? 보안이 강화하기 위해. 네트워크 관리자는 세분화된 정책에 따라 서브넷 간의 트래픽 흐름을 제어 모니터링과 성능을 개선.
            - 예시? 트러스트 가정, 제로 트러스트 대응
                - 트러스트 가정? 예전에는 외부 세계와 회사 사업에 중요한 데이터를 분리하는 네트워크 경계에 보안 전략을 적용했음. 경계 내에 있는 사람은 신뢰할 수 있고, 위협이 아니라고 가정했기 때문. 따라서 정보 액세스 권한에 제한이 거의 없었음. 그러나 최근 들어 대규모 해킹 사건으로 인해 트러스트 가정에 대한 확신이 무너졌는데, 이는 내부자가 보안 침해의 원인이 될 수 있기 때문.
                - 제로 트러스트 대응? 기본적으로 모든 사람, 심지어 이미 네트워크 경계 내에 있는 사람조차 신뢰할 수 없다고 가정.  "보호 표면"을 중심으로 작동하는데 이는 전체 네트워크 경계의 공격 표면보다 훨씬 작음. 여기에 네트워크 세그먼테이션이 필요. 세그먼테이션을 활용하여 보호 표면 주변으로 마이크로 경계를 구성하고 제2의 방어선을 구축하므로써 권한이 있는 사용자가 보호 표면 내의 자산에 액세스하고 그 외의 사용자는 기본적으로 차단할 수 있음.
    - 데이터 전송 단위 : 패킷, 데이터그램
    - 대표 프로토콜 : IP, ICMP, ARP

4. 전송 계층
    - 정의 : 양 끝단(End to end)의 사용자들이 신뢰성있는 데이터를 주고 받을 수 있도록 해 주어, 상위 계층들이 데이터 전달의 유효성이나 효율성을 생각하지 않도록 해주는 계층
    - 역할 : TCP와 UDP 프로토콜을 통해 통신을 활성화한다. 포트를 열어두고, 프로그램들이 전송을 할 수 있도록 제공.
        - 방법? 프로세스 수준 주소 지정, '다중화와 역다중화', '단편화/패키징/재조합', 연결 수립/유지/종료, 승인 및 재전송, 흐름 제어
            - 다중화와 역다중화?
            - 단편화?
            - 패키징?
            - 재조합?
    - 대표 프로토콜 : TCP, UDP
    - 데이터 전송 단위 : Segment

5. 세션 계층
    - 역할 : 데이터가 통신하기 위한 논리적 연결을 담당한다. TCP/IP 세션을 만들고 없애는 책임을 지니고 있다.
        - 방법 : 세션 생성/유지/종료, 인터페이스. 
    - 데이터 전송 단위 : 세션
    - 대표 프로토콜 : 소켓, 그리고 NetBIOS, RPC, 

6. 표현 계층
    - 역할 : 데이터 표현에 대한 독립성을 제공하고 암호화하는 역할을 담당.
        - 방법 : 파일 인코딩, 명령어를 포장, 압축, 암호화
    - 데이터 전송 단위 : 인코딩 데이터
    - 대표 프로토콜 : MIME, SSL

7. 응용 계층
    - 역할 : 응용 프로세스와 직접 관계하여 사용자 인터페이스, 전자우편, 데이터베이스 관리 등의 서비스를 제공.
    - 데이터 전송 단위 : 메세지, 사용자 데이터
    - 대표 프로토콜 : HTTP, FTP, SMTP, DHCP, NFS, Telnet, SNMP, POP3

### TCP와 UDP
1. TCP
인터넷 상에서 데이터를 메세지의 형태 죽,세그먼트 라는 블록 단위로 보내기 위해 IP와 함께 사용하는 프로토콜이다.
TCP와 IP를 함께 사용하는데, IP가 데이터의 배달을 처리한다면 TCP는 패킷을 추적 및 관리한다.
높은 신뢰성을 보장한다.
연속성보다 신뢰성있는 전송이 중요할 때에 사용된다.
    - 3 way handshake? TCP 통신을 하기 위하여 클라이언트와 서버 간에 SYN, SYN/ACK, ACK 세 번의 트랜잭션을 주고 받아서 연결을 맺는 과정
        - 과정?
            1) 클라이언트 -> 서버 : SYN
                클라이언트가 서버에게 임의의 SEQ 넘버로 SYN 플래그가 설정된 패킷을 송신한다. (일반적으로 connect() 함수를 호출한다.)
                PORT 상태 : 클라이언트(SYN_SENT), 서버(LISTEN)
            2) 서버 -> 클라이언트 : SYN + ACK
                서버는 SEQ 넘버를 랜덤, ACK 넘버를 수신한 SEQ + 1로 지정하고, SYN과 ACK 플래그를 설정한 패킷을 송신한다.
                PORT 상태 : 서버(SYN_RCVD), 클라이언트(SYN_SENT)
            3) 클라이언트 -> 서버 : ACK
                클라이언트는 SEQ 넘버를 수신한 ACK, ACK 넘버를 수신한 SEQ + 1로 지정하고, ACK 플래그를 설정한 패킷을 송신한다.
                PORT 상태 : 클라이언트(ESTABLISHED), 서버(ESTABLISHED)
    - 4 way handshake? TCP 연결을 안정적으로 해제하기 위하여 클라이언트와 서버 간에 FIN, ACK, FIN, ACK 네 번의 트랜잭션을 주고 받는 과정
        - 과정?
            1) 클라이언트 -> 서버 : FIN
                클라이언트가 서버에게 FIN 플래그가 설정된 패킷을 송신한다. (일반적으로 close() 함수를 호출한다.)
                PORT 상태 : 클라이언트(FIN_WAIT1), 서버(ESTABLISHED)
            2) 서버 -> 클라이언트 : ACK
                서버가 클라이언트에게 ACK 플래그가 설정된 패킷을 송신한다.
                PORT 상태 : 서버(CLOSE_WAIT), 클라이언트(FIN_WAIT2)
            3) 서버 -> 클라이언트 : FIN
                서버가 클라이언트에게 FIN 플래그가 설정된 패킷을 송신한다. (일반적으로 close() 함수를 호출한다.)
                PORT 상태 : 서버(LAST_ACK), 클라이언트(FIN_WAIT2)
            4) 클라이언트 -> 서버 : ACK
                클라이언트가 서버에게 ACK 플래그가 설정된 패킷을 송신한다.
                PORT 상태 : 클라이언트(TIME_WAIT), 서버(CLOSED)

2. UDP
데이터를 데이터그램 단위로 처리하는 프로토콜이다.
정보를 주고 받을 때 정보를 보내거나 받는다는 신호절차를 거치지 않는다.

3. TCP vs UDP
| TCP | UDP |
| --- | --- |
| 애플리케이션이 데이터를 안정적으로 송수신할 수 있도록 하는 프로토콜 | 단순하고 빠르게 애플리케이션이 네트워크 계층에 접근할 수 있도록 하는 인터페이스를 제공하는 프로토콜 |
| 연결형 서비스가상 회선 방식 제공 | 비연결형 서비스로 데이터그램 방식 제공 |
| 높은 신뢰성 보장 | 신뢰성 낮아 연속성이 중요한 서비스에 사용 |
| 스트림 기반으로 데이터의 경계(boundary)가 없다 | 메시지 기반으로 데이터의 경계(boundary)가 있다 |
| 모든 데이터 전송을 관리하며, 손실된 데이터는 자동으로 재전송 | 자체적인 재전송 기능 없음 |
